# Security Policy

## Reporting a Vulnerability

If you discover a security issue in the Principia Fractalis repository or evidence chain, please contact us immediately.

**Contact:** security@fractaldevteam.org (or create a private security advisory on GitHub)

### Response Timeline
- We will acknowledge receipt within **48 hours**
- We will provide an initial assessment within **7 days**
- We will work with you to understand and address the issue

### What to Include
- Description of the vulnerability
- Steps to reproduce
- Potential impact on evidence integrity
- Any suggested fixes

### Scope
This repository contains mathematical proofs and evidence for the Principia Fractalis framework. Security concerns include:
- Integrity of mathematical proofs and evidence files
- Provenance and authenticity of commits
- Protection against tampering or unauthorized modifications

### Out of Scope
- Theoretical challenges to the mathematical framework (use public issues for academic discussion)
- Typos or formatting issues (use regular pull requests)

## Security Measures in Place
- Branch protection on main with required reviews
- Signed commits required for provenance
- Automated dependency scanning (Dependabot)
- Code scanning (CodeQL)
- Secret scanning and push protection

## Disclosure Policy
We follow responsible disclosure practices. We request that you:
- Do not publicly disclose the vulnerability until we've had a chance to address it
- Make a good faith effort to avoid privacy violations and data destruction
- Contact us before performing any destructive testing

Thank you for helping keep Principia Fractalis and its evidence chain secure.
